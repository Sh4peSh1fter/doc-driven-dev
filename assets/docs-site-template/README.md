# Docs site

A [Docusaurus](https://docusaurus.io/) site that publishes the repository's `docs/` tree (one level up). Generated by doc-driven-dev (`/docs-site init`).

## Develop

```bash
npm install
npm start          # local dev server with hot reload
```

## Build

```bash
npm run build      # static build into ./build; fails on broken links
npm run serve      # serve the production build locally
```

## How it works

- Content comes from `../docs` (`docs.path` in `docusaurus.config.ts`).
- The sidebar is autogenerated from the folder structure and each section's `_category_.json`.
- Underscore-prefixed files/folders, `_tasks/`, `_templates/`, `_assets/`, and `99-archive/` are excluded.
- Broken links fail the build (`onBrokenLinks: "throw"`), matching the docs validator's link gate.

Run `node scripts/find-missing-docs.mjs` to list enabled sections that still need content.
